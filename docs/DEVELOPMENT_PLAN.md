# План развития (соло, 2–3 ч/день + Cursor)

Режим: вы — разработчик, проектировщик тестовых данных и владелец продукта; **пока нет пользователей** — закладываем фундамент без срочных «пожаров».

## Ритм

- **Один главный фокус в сессии** (2–3 ч): довести до коммита + при необходимости строка в `CHANGELOG.md` или в этом файле в разделе «Журнал».
- **Микро-релиз раз в 1–2 недели**: bump `VERSION`, короткий список изменений.
- **Cursor**: в начале сессии — цель и критерий готовности; в конце — что сделано (чтобы не терять контекст).

## Сборка (артефакты)

Из корня репозитория:

```bat
build.bat
```

Скрипт ставит `build` и `wheel`, затем `python -m build --no-isolation` (без отдельного venv — **не требует скачивать setuptools с PyPI** при каждой сборке; нужен нормальный интернет только для первого `pip install build wheel`).

Результат: каталог `dist/` (в `.gitignore`, в git не коммитится) — `.whl` и `.tar.gz`. Установка в чистое venv:

```bat
pip install dist\nordfox_raskroy-*-py3-none-any.whl
nordfox-raskroy
```

Полная изоляция (если нужна воспроизводимость как на CI): `python -m build` без флага — потребует сеть для зависимостей изоляции.
